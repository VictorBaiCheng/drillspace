# DrillSpace V2.8.9：MyDrill DLL 对照校准版

本版目标：不再只看前端，而是建立“旧 MyDrill/well-path DLL 输出结果”与“DrillSpace FastAPI Python 轨迹引擎”的逐列对照机制。

## 一、现在能做什么

### 1. 轨迹算法对照
把旧 MyDrill 导出的轨迹结果 CSV 输入 V2.8.9，系统会自动：

- 读取 MyDrill 结果：MD / INC / AZI / TVD / NS / EW / DOGLEG / BUILD / TURN；
- 用 FastAPI `minimum_curvature` 引擎重新计算；
- 按 MD 对齐测点；
- 逐列计算误差；
- 输出 maxAbs / meanAbs / RMSE；
- 判断 PASS / REVIEW。

### 2. 防碰算法后续对照
本版已经保留 V2.8.7 的防碰参考引擎。DLL 输出的防碰结果拿到后，也可以按同样方式扩展：

- 最近距离；
- 分离距；
- 分离系数；
- 法平面角；
- 水平扫描角；
- 误差椭球参数。

## 二、启动接口

启动后端：

```bat
cd /d D:\WCode\DrillSpace\drillspace\backend
python -m uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

检查：

```text
http://127.0.0.1:8000/api/health
http://127.0.0.1:8000/api/well-path/engine-info
http://127.0.0.1:8000/api/well-path/calibration/template
http://127.0.0.1:8000/api/well-path/calibration/sample
```

## 三、命令行对照

把 MyDrill 导出的 CSV 放到：

```text
backend/sample_data/mydrill_reference_export.csv
```

执行：

```bat
cd /d D:\WCode\DrillSpace\drillspace\backend
python scripts\compare_mydrill_reference.py --reference sample_data\mydrill_reference_export.csv --out data\calibration\report.json
```

## 四、重要说明

`cal_wellbore.dll` 是 JNI DLL，不是普通 C ABI DLL。Python 不能直接用 ctypes 简单调用。
所以 V2.8.9 先采用“旧系统导出CSV结果 → FastAPI逐列对照”的工程路线。

后续 V2.9.0 可以继续做 Java Bridge，直接加载 DLL 并批量输出 JSON/CSV。
